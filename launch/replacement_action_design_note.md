# BreakWave — Healthy Replacement Action Design Note

## Purpose

BW-63A confirms that the healthy replacement action selector stays single-select for MVP.

## Decision

Healthy replacement action is intentionally one primary next move.

## Reason

In a high-pressure urge moment, BreakWave should help the user choose one clear next action instead of turning the log into a long checklist.

## User model

- Healthy replacement action = the main intended replacement move.
- Action taken = what actually happened, including multiple actions if needed.

Example:

- Healthy replacement action: Open Rescue
- Action taken: Opened Rescue, left the room, and texted someone safe.

## Future option

A later Plus feature may allow a larger replacement-action library or optional multi-action review, but MVP should stay focused and fast.
